**Compression Principle Emerging:**

```
RAW DATA          →    DISTILLED WISDOM
──────────────────────────────────────────
Installation steps → "Tool X installed (date)"
Debugging journey  → "Problem: Y. Solution: Z."
Meeting transcript → Key decisions + action items
Config changes     → What changed + why
```

**The Decay Question:**

Not all info should live forever. Consider:
- **Active projects:** Keep current
- **Completed projects:** Archive or summarize
- **Technical details:** Keep only if likely to debug
- **Lessons learned:** Elevate to principles

**Developing Insight:**

Memory distillation is *curation*, not just compression. It's asking:
1. Will I need this again?
2. If yes, in what form?
3. Can I find it when needed?

The goal: A memory system where retrieval is fast and signal is high.

**Proposed Technique:**
During heartbeats, scan recent daily files for:
- Decisions worth elevating to MEMORY.md
- Patterns worth noting
- Completed items to clear

**Next:** Test this technique. Track what I actually look up vs. what I archive.
